# Global Development Standards

## 1. Naming Conventions

- Use meaningful and descriptive names
- Variable names should clearly represent purpose
- Function names should describe actions
- Use consistent naming across project
- Avoid unclear abbreviations
- Keep names concise and readable

Examples:
- good: customerDetails
- bad: cd

---

## 2. Error Handling

- Handle all possible exceptions properly
- Use try-catch blocks where necessary
- Display user-friendly error messages
- Log errors for debugging
- Prevent application crashes

---

## 3. Accessibility

- Ensure keyboard accessibility
- Use semantic HTML
- Add aria labels where required
- Maintain proper color contrast
- Support responsive design

---

## 4. Configuration Driven Development

- Use custom labels/configuration instead of hardcoded values
- Store reusable settings centrally
- Use environment variables for secrets

---

## 5. Code Comments

- Add comments for complex logic
- Keep comments meaningful and updated
- Avoid unnecessary comments

---

## 6. Code Formatting and Indentation

- Maintain proper indentation
- Follow consistent formatting
- Keep code clean and readable

---

## 7. Unit Testing

- Generate unit tests for critical functionality
- Cover edge cases
- Maintain high test coverage

---

## 8. Best Practices

- Use reusable components
- Follow modular architecture
- Optimize performance
- Avoid duplicate code
- Follow security standards
- Ensure maintainability and scalability

---

## 9. UI Standards

- Use responsive design
- Use reusable UI components
- Follow branding-rules.md
- Maintain consistent spacing and typography
